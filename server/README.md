# Huestima Backend

Persistent Node.js multiplayer backend for Huestima. It runs separately from the Next.js frontend and is designed for a VPS process behind Caddy or Nginx.

## Local Development

```bash
cd server
npm install
cp .env.example .env
npm run dev
```

The local backend defaults to `http://localhost:4000`.

Set the frontend env in the project root:

```bash
NEXT_PUBLIC_SOCKET_URL=http://localhost:4000
```

Then run the frontend:

```bash
npm run dev
```

## Health Checks

```bash
curl http://localhost:4000/
curl http://localhost:4000/health
```

`/health` returns:

```json
{
  "ok": true,
  "uptime": 123,
  "rooms": 1,
  "activeSockets": 2,
  "timestamp": "2026-05-22T09:00:00.000Z"
}
```

## Environment

```bash
NODE_ENV=production
PORT=4000
CLIENT_ORIGINS=https://huestima.com,https://www.huestima.com
ROOM_TTL_MS=3600000
EMPTY_ROOM_TTL_MS=120000
DISCONNECT_GRACE_MS=45000
HOST_DISCONNECT_GRACE_MS=60000
MAX_PLAYERS_PER_ROOM=5
```

`CLIENT_ORIGINS` is a comma-separated whitelist used by both Express CORS and Socket.IO.

## Socket Contract

All important client actions use acknowledgement callbacks:

```json
{
  "ok": true,
  "data": {}
}
```

Errors use:

```json
{
  "ok": false,
  "error": "Clean message"
}
```

Client events:

- `room:create`
- `room:join`
- `room:getState`
- `room:leave`
- `room:kickPlayer`
- `room:startGame`
- `game:submitGuess`
- `game:submitResults`

Compatibility aliases are also supported:

- `room:requestState`
- `game:start`
- `round:requestTarget`
- `round:submitGuess`

Server events:

- `room:state`
- `room:created`
- `room:joined`
- `room:playerKicked`
- `room:closed`
- `game:started`
- `game:submissionReceived`
- `game:scoreboard`
- `game:leaderboard`
- `connection:restored`

## Game Rules

- Room codes are exactly 6 digits.
- Rooms hold up to 5 players.
- Only the host can start the game.
- Only the host can kick other lobby players.
- The host cannot kick themselves.
- Multiplayer target colors are generated by the server from a room seed.
- Scores are calculated server-side with the same OKLab distance model as singleplayer.
- Rounds score out of 10, total score is out of 50.
- The scoreboard is broadcast only after all active players finish.
- Disconnected players have a grace period before they stop blocking the scoreboard.

## Hetzner / Ubuntu Deployment

Clone or pull the repo on the VPS:

```bash
cd /var/www/huestima-app/server
npm install --omit=dev
cp .env.example .env
nano .env
```

Start directly:

```bash
npm run start
```

Or run with PM2:

```bash
pm2 start src/index.js --name huestima-backend
pm2 save
pm2 startup
```

Useful PM2 scripts:

```bash
npm run pm2:start
npm run pm2:restart
npm run pm2:logs
```

## Caddy Reverse Proxy

Example `Caddyfile`:

```caddyfile
api.huestima.com {
  reverse_proxy localhost:4000
}
```

Set Vercel frontend env:

```bash
NEXT_PUBLIC_SOCKET_URL=https://api.huestima.com
```

Socket.IO uses websocket and polling transports, so it works behind normal Caddy or Nginx reverse proxy setups.
